使用版本控制工具Git上传 代码至Github

Created2024-05-12|Updated2024-05-11
|Post Views:

git add . 添加当前文件夹下面的所有文件到.git文件夹中仓库的缓存区中

git status 查看当前状态

git commit -m “” 引号中添加更新的注释

git pull 拉取当前远端文件仓库中的文件，检查是否冲突

git push 将更新后的代码上传至远端仓库中去

git rm –cached *.docx 删除已经跟踪过的文件，标记为未跟踪

  1. 新建分支
    首先，在你新建分支之前，你应该在master分支，并且要保证你当前的是最新代码，要不然就是找死，最后提交代码就会出问题。

git pull origin master ，在master分支下，保证当前代码与线上同步，是最新的
git branch <分支名> ，正儿八经的新建分支。
git checkout <分支名> ，切换 到新建的分支上，再进行下一步。
4. git push origin <分支名> ,把本地分支推到远端，让远端也有一个你的分支，用来后面提交你的代码。
新建分支并切换到该分支：git checkout -b <分支名>

  1. 提交代码
    在新建的分支上写完代码之后，就可以提交代码了。此时应该在你新建的分支上

git status ,查看自己写了哪些东西。
git add .
git commit -m
